Overview

This guide is for operating PhoneDukaan in production, fixing common failures quickly, maintaining both frontend and backend safely, and rebuilding the whole stack on a new VPS if required.

Production Domains

  • Frontend: https://phonedukaan.kohnex.com
  • Admin API: https://adminapi.kohnex.com
  • PocketBase: https://db.kohnex.com

Current Working Model

  • Shop app: browser talks directly to PocketBase for shop data
  • Admin panel: browser talks to admin API only
  • Admin API: server talks to PocketBase using superuser auth
  • Password reset: PocketBase sends email using its own app URL
Frontend live on Netlify
Admin API behind Nginx
systemd service: adminapi
TLS active on both backend domains

Full System Architecture

The system is split into a Netlify-hosted frontend and a VPS-hosted backend stack. Shop users and admin users follow different data and authentication paths.

Frontend: Netlify

Vite + React app served from Netlify. Routes include normal shop usage on / and admin access on /777admin.

  • Shop session: stored in localStorage
  • Admin session: stored in sessionStorage
  • PWA: service worker enabled via Vite PWA plugin

Admin API: VPS

Node-based API on adminapi.kohnex.com. It accepts admin login from the frontend, validates bearer tokens, and performs trusted admin actions against PocketBase.

  • Service: adminapi.service
  • Port: 127.0.0.1:8787
  • Proxy: Nginx reverse proxy

PocketBase: VPS

PocketBase is the main data store and auth service for shop users. It also sends password reset emails and stores collections for all business data.

  • Main host: db.kohnex.com
  • Important setting: app URL must be https://db.kohnex.com
Browser -> Netlify frontend -> Admin API -> PocketBase

Frontend Architecture

  • Main files: mobile-dukaan_2.jsx, pocketbase-client.js, vite.config.js
  • Build output: generated into dist/
  • SPA rewrite: public/_redirects for Netlify route fallback
  • Admin path: direct loads to /777admin now work

Backend Architecture

  • Admin server: admin-api-server.js
  • Nginx: handles public HTTPS and proxies to local ports
  • systemd: keeps the Node admin API running after reboot
  • Certbot: manages certificates

Core PocketBase Collections

  • shop_users
  • shops
  • app_settings
  • inventory
  • transactions
  • photos
Layer Host Purpose Auth Model
Frontend phonedukaan.kohnex.com Shop app and admin UI Shop sessions in localStorage, admin token in sessionStorage
Admin API adminapi.kohnex.com Trusted admin actions, dashboard, user management, settings, reset email trigger Bearer token with in-memory session store
PocketBase db.kohnex.com Database, shop auth, business records, reset email generation Shop auth via shop_users; admin superuser only used server-side

How The System Works

Use this section to understand the live data flow before making changes. It is the reference model for what is supposed to happen when the system is healthy.

Shop User Flow

1. LoginUser signs in with mobile number + password. The browser authenticates directly to PocketBase using the shop_users collection.
2. Session restoreShop session is stored in localStorage. On reload, the app restores the saved shop session.
3. Data accessInventory, transactions, photos, and shop profile are loaded directly from PocketBase.
4. Realtime syncThe app subscribes to PocketBase realtime updates for shop data outside the admin route.

Admin Flow

1. Admin loginThe browser sends credentials to https://adminapi.kohnex.com/admin-api/login.
2. Token issueThe admin API authenticates against PocketBase superuser server-side and returns a bearer token to the browser.
3. Session validationThe browser stores the token in sessionStorage and validates it via /admin-api/session.
4. Protected actionsDashboard load, user updates, shop updates, settings changes, and password reset requests all go through the admin API only.

Password Reset Flow

  • Admin panel sends a request to /admin-api/password-reset
  • Admin API calls PocketBase requestPasswordReset(email)
  • PocketBase sends the email and generates the reset link
  • The email link host depends on PocketBase Application URL
  • Correct value: https://db.kohnex.com

Routing + Cache Behavior

  • Netlify serves the SPA frontend
  • public/_redirects sends all routes to index.html
  • This is why direct loads to /777admin work
  • PWA service worker can cache old assets, so use hard refresh when debugging

Frontend Maintenance

Use this process whenever you change the UI, admin flow, PocketBase client code, or Netlify routing.

Local Build Verification

Before any deploy, install dependencies and build the frontend to catch syntax or bundling problems.

npm install
npm run build

Frontend Files That Matter

  • mobile-dukaan_2.jsx - main app and admin UI
  • pocketbase-client.js - all PocketBase and admin API requests
  • vite.config.js - build + PWA config
  • public/_redirects - Netlify SPA routing
  • .env.example - environment variable reference

Netlify Environment Variables

  • VITE_POCKETBASE_URL=https://db.kohnex.com
  • VITE_ADMIN_API_URL=https://adminapi.kohnex.com
  • After changing env vars, trigger a new deploy
  • Do not put secrets in frontend env vars

Frontend Deploy Checklist

1. Update code locally
2. Run npm run build
3. Push code to GitHub
4. Wait for Netlify deploy to finish
5. Open https://phonedukaan.kohnex.com
6. Open https://phonedukaan.kohnex.com/777admin
7. Test login, dashboard, logout
8. Check browser Network tab if anything looks wrong

Clear Frontend Cache Problems

1. Open site in Incognito
2. Hard refresh the page
3. In DevTools, check "Disable cache"
4. If still stale, unregister service worker in Application tab
5. Reload again

Backend Maintenance

The backend includes Nginx, PocketBase, the admin API, TLS certificates, and systemd service management.

Admin API Service

  • Service name: adminapi
  • File: /etc/systemd/system/adminapi.service
  • Working directory: /var/www/phonedukaan
  • Port: 127.0.0.1:8787

Nginx Reverse Proxy

  • PocketBase host: db.kohnex.com
  • Admin API host: adminapi.kohnex.com
  • Nginx terminates HTTPS and proxies to local services
  • Always run nginx -t before reload

Admin API Health Commands

systemctl status adminapi --no-pager
journalctl -u adminapi -n 100 --no-pager
systemctl restart adminapi
ss -ltnp | grep 8787

Nginx + TLS Health Commands

PocketBase Checks

  • Confirm PocketBase is reachable on https://db.kohnex.com
  • Confirm Application URL is https://db.kohnex.com
  • Check collections and data in admin dashboard if issues appear
  • Verify SMTP settings if password reset stops working

Admin API Environment Values

  • POCKETBASE_URL=https://db.kohnex.com
  • ADMIN_API_PORT=8787
  • ADMIN_API_ALLOWED_ORIGINS=https://phonedukaan.kohnex.com

Error Resolution Guide

Use these fixes in order. Start with the symptom, run the matching commands, and only make changes after you know where the failure is.

Problem Likely Cause Resolution
/777admin shows Netlify 404 SPA route fallback missing Ensure public/_redirects exists with /* /index.html 200, then redeploy frontend.
Admin login fails Admin API down, CORS mismatch, wrong env var, wrong credentials Check systemctl status adminapi, validate VITE_ADMIN_API_URL, inspect browser network requests.
Dashboard does not load Admin token invalid or backend query failure Test /admin-api/session and /admin-api/dashboard manually with curl.
502 Bad Gateway Nginx cannot reach backend process Check service health, logs, and listening port. Restart the failed service.
SSL hostname mismatch Wrong certificate attached to Nginx vhost Check certbot certificates and the exact server_name in Nginx, then reissue correct certificate.
Password reset email points to localhost PocketBase Application URL is wrong Set PocketBase Application URL to https://db.kohnex.com and resend reset email.
Direct refresh shows old UI Old service worker or browser cache Hard refresh, disable cache in DevTools, or unregister the service worker.
CORS errors in admin panel Origin missing from allowed list Update ADMIN_API_ALLOWED_ORIGINS, restart admin API, retest.

Backup Plan

A working migration depends on having current backups of both code and PocketBase data. Without PocketBase data, restoring only the frontend is not enough.

What Must Be Backed Up

  • GitHub repository
  • PocketBase data directory
  • PocketBase uploaded files
  • Nginx site configs
  • /etc/systemd/system/adminapi.service
  • DNS notes and domain mapping
  • Netlify environment values
  • Admin API environment values

Recommended Backup Frequency

  • Code: every change via GitHub
  • PocketBase data: daily
  • Before risky updates: manual backup first
  • Before VPS migration: fresh backup immediately before cutover

Backup Checklist

1. Backup PocketBase database/data directory
2. Backup PocketBase uploaded files
3. Backup /etc/nginx/sites-available/
4. Backup /etc/nginx/sites-enabled/
5. Backup /etc/systemd/system/adminapi.service
6. Confirm GitHub repo is up to date
7. Save Netlify env vars in a secure place
8. Save DNS records and server IP notes

VPS Crash Recovery and Migration Plan

If the current VPS crashes, use this exact order. Do not move DNS first. Restore the stack fully, test it, and only then point domains to the new server.

Phase 1: Prepare New VPS

apt update
apt install -y nginx certbot python3-certbot-nginx curl git
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
node -v
npm -v

Phase 2: Restore Application Code

mkdir -p /var/www/phonedukaan
cd /var/www/phonedukaan
git clone YOUR_GITHUB_REPO_URL .
npm install

Phase 3: Restore PocketBase

1. Restore PocketBase binary/service
2. Restore PocketBase data directory
3. Restore uploaded files
4. Start PocketBase
5. Confirm https://db.kohnex.com will proxy correctly after Nginx is restored

Phase 4: Restore Admin API Service

cat >/etc/systemd/system/adminapi.service <<'EOF'
[Unit]
Description=PhoneDukaan Admin API
After=network.target

[Service]
Type=simple
WorkingDirectory=/var/www/phonedukaan
Environment=POCKETBASE_URL=https://db.kohnex.com
Environment=ADMIN_API_PORT=8787
Environment=ADMIN_API_ALLOWED_ORIGINS=https://phonedukaan.kohnex.com
ExecStart=/usr/bin/node /var/www/phonedukaan/admin-api-server.js
Restart=always
RestartSec=5
User=root

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable adminapi
systemctl start adminapi
systemctl status adminapi --no-pager

Phase 5: Restore Nginx

1. Restore Nginx site config for db.kohnex.com
2. Restore Nginx site config for adminapi.kohnex.com
3. Run nginx -t
4. Run systemctl reload nginx

Phase 6: Issue SSL Certificates

certbot --nginx -d db.kohnex.com
certbot --nginx -d adminapi.kohnex.com
certbot certificates

Phase 7: Test Before DNS Cutover

curl -i http://127.0.0.1:8787/admin-api/session
curl -i https://adminapi.kohnex.com/admin-api/session \
  -H "Origin: https://phonedukaan.kohnex.com"
curl -i https://db.kohnex.com
systemctl status adminapi --no-pager
systemctl status nginx --no-pager
certbot certificates

Phase 8: Move DNS to New VPS

1. Update A record for db.kohnex.com to new VPS IP
2. Update A record for adminapi.kohnex.com to new VPS IP
3. Wait for DNS propagation
4. Re-test both domains
5. Confirm frontend still points to correct hosts

Phase 9: Final Post-Migration Checks

1. Open https://phonedukaan.kohnex.com
2. Open https://phonedukaan.kohnex.com/777admin
3. Log into admin panel
4. Confirm dashboard loads
5. Send password reset email
6. Confirm reset link uses https://db.kohnex.com
7. Test one user/shop update
8. Restart VPS and confirm services auto-start
